# BOM 成本对齐 Skill

## 触发条件

当用户说以下关键词时触发：

- "BOM成本核算" / "对齐BOM" / "BOM匹配采购" / "自动算BOM价格"
- "把采购价填到BOM里" / "生成成本核算表"
- 或在项目文件夹语境中要求"处理BOM"

## 功能概述

输入 **原始 BOM + 采购成本表**，输出 **手工版格式的 BOM 成本核算 Excel**。

## 输入文件

### 必须文件（缺一不可）

1. **原始 BOM 表**（`{项目号}.xlsx`）— 统一22列布局
   - `机械BOM表` sheet：零件行带 `件号`（含 `P` 后缀）+ 组件标题行（`（组件名）×N`）
   - `电气BOM表` sheet：电气零件用 `E{项目号}-{序号}` 格式；栏目分段用名称标题行表示，例如 `伺服/辊道电机`、`测量部分`、`电器柜` 等

2. **采购成本表**（含 `明细` sheet）
   - 数据从第 4 行开始
   - col[0]：`图号/零件号`，col[8]：`含税金额`

## 输出文件

`{项目号}_BOM成本核算_自动生成.xlsx`，含两个 sheet：

| 列 | 标题 | 内容 |
|---|---|---|
| A | 件号 | 零件号、组件标题或电气栏目标题 |
| B | 零件名称 | 组件名、栏目名或零件名 |
| C | 型号规格 | 规格型号 |
| D | 说明 | 材料/采购类型 |
| E | 数量 | BOM 用量 |
| F | 品牌 | 品牌 |
| G | 单价 | 采购含税金额 ÷ BOM数量 |
| H | 价格 | 采购含税金额 |
| I | 组件数量 | 仅机械组件行：标题中 ×N 的数量 |
| J | 合计总价 | 机械组件行：零件价之和 × 组件数量；电气栏目行：栏目下属零件价之和 |

### 格式

- 表头：灰底加粗，下边框加粗
- 机械组件行：浅蓝底色（#DAEEF3）
- 电气栏目行：浅蓝底色（#DAEEF3）
- 全表细线边框

## 分段规则

### 机械BOM

机械按组件标题行分段。组件标题行通常为 `（组件名）×N`，脚本会从标题中提取组件数量，并在 J 列写组件合计。

### 电气BOM

电气按名称栏目行分段。栏目行只作为分段标题，不参与采购匹配、不计数量、不当作零件。

电气栏目行识别规则：

1. A-H 范围内只有一个非空文本；
2. 该文本不是电气零件号（不匹配 `^E\d+-\d+$`）；
3. 该文本不是机械/设备组件标题（不包含 `（...）×N` 这类组件格式）。

例如：`伺服/辊道电机`、`测量部分`、`电器柜`、`吊柜`、`安全元件`、`现场电气元件`。

## 匹配公式

```
BOM单价 = 采购含税金额 ÷ BOM数量
价格 = 采购含税金额
机械组件合计 = Σ(下属零件价格) × 组件数量
电气栏目小计 = Σ(该栏目下属电气零件价格)
```

## 脚本

`scripts/bom-cost-align.py` — 独立 Python 脚本，用法：

```bash
# 自动查找：默认从当前目录下的项目文件夹读取
python3 scripts/bom-cost-align.py PROJECT_ID

# 指定项目根目录，只统计不输出
python3 scripts/bom-cost-align.py PROJECT_ID --base /path/to/projects --dry-run

# 直接指定文件
python3 scripts/bom-cost-align.py project --bom BOM.xlsx --cost cost.xlsx
```

## 已知限制

1. **件号前缀不匹配**：采购文件与 BOM 的件号前缀不一致时，需人工确认映射规则
2. **不含调价/利润**：纯采购成本基准
3. **只用采购数据**：不使用 BOM 原始单价

## 最小验收

用代表项目跑一次：

```bash
python3 skills/bom-cost-align/scripts/bom-cost-align.py PROJECT_ID --base /path/to/projects --dry-run
python3 skills/bom-cost-align/scripts/bom-cost-align.py PROJECT_ID --base /path/to/projects
```

验收点：

- dry-run 输出机械/电气匹配数量与合计金额
- 输出文件存在
- `电气BOM` sheet 能看到栏目行，栏目行浅蓝底色，且不计入零件匹配数量
