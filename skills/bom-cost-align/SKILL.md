# BOM 成本对齐 Skill

## 触发条件

当 Kevin 说以下关键词时触发：

- "BOM成本核算" / "对齐BOM" / "BOM匹配采购" / "自动算BOM价格"
- "把采购价填到BOM里" / "生成成本核算表"
- 或在项目文件夹语境中要求"处理BOM"

## 功能概述

输入 **原始 BOM + 采购成本表**，输出 **手工版格式的 BOM 成本核算 Excel**。

支持三种项目形态：

1. 机械 + 电气：同时解析 `机械BOM表` 与 `电气BOM表`，输出两个 sheet。
2. 纯机械：只存在 `机械BOM表` 时，输出 `机械BOM` sheet，跳过电气。
3. 纯电气：只存在 `电气BOM表` 时，输出 `电气BOM` sheet，跳过机械。

## 输入文件兼容规则

### BOM 文件

自动查找时优先支持：

- `{项目号}.xlsx`
- 文件名包含项目号且包含 `BOM`，例如 `{项目号}_BOM.xlsx`、`{项目号}_整机BOM.xlsx`

BOM workbook 至少包含下列 sheet 之一：

- `机械BOM表`：零件行带 `件号`（含 `P` 后缀）+ 组件标题行（`（组件名）×N`）
- `电气BOM表`：电气零件用 `E{项目号}-{序号}` 格式；栏目分段用名称标题行表示，例如 `伺服/辊道电机`、`测量部分`、`电器柜` 等

如果两张 BOM sheet 都不存在，则停止处理并报告缺失。

### 成本表

自动查找时支持文件名包含项目号且包含：`采购成本`、`成本核算`、`成本`。

成本表必须包含零件级价格明细。脚本按表头识别，不强依赖固定 sheet 名：

- 标准采购明细：识别 `图号/零件号` / `零件编号` / `件号` 与 `含税金额`。
- 成本化 BOM 表：识别 `件号` / `零件编号`，并优先使用 `总价`；没有总价时用 `总数 × 汇率 × 物料含税单价 + 加工费用总价`。

只有品牌、产品、采购金额汇总而没有零件号的成本表不能用于零件级匹配，脚本应明确报错，不生成误导性成本文件。

## 输出文件

`{项目号}_BOM成本核算_自动生成.xlsx`，按原始 BOM 中实际存在的类型输出 sheet：

- 有 `机械BOM表` → 输出 `机械BOM`
- 有 `电气BOM表` → 输出 `电气BOM`
- 缺失的类型不输出空 sheet

| 列 | 标题 | 内容 |
|---|---|---|
| A | 件号 | 零件号、组件标题或电气栏目标题 |
| B | 零件名称 | 组件名、栏目名或零件名 |
| C | 型号规格 | 规格型号 |
| D | 说明 | 材料/采购类型 |
| E | 数量 | BOM 用量 |
| F | 品牌 | 品牌 |
| G | 单价 | 采购含税金额 ÷ BOM数量 |
| H | 价格 | 采购含税金额 |
| I | 组件数量 | 仅机械组件行：标题中 ×N 的数量 |
| J | 合计总价 | 机械组件行：零件价之和 × 组件数量；电气栏目行：栏目下属零件价之和 |

## 分段规则

### 机械BOM

机械按组件标题行分段。组件标题行通常为 `（组件名）×N`，脚本会从标题中提取组件数量，并在 J 列写组件合计。

### 电气BOM

电气按名称栏目行分段。栏目行只作为分段标题，不参与采购匹配、不计数量、不当作零件。

电气栏目行识别规则：

1. A-H 范围内只有一个非空文本；
2. 该文本不是电气零件号（不匹配 `^E\d+-\d+$`）；
3. 该文本不是机械/设备组件标题（不包含 `（...）×N` 这类组件格式）。

## 匹配公式

```
BOM单价 = 采购含税金额 ÷ BOM数量
价格 = 采购含税金额
机械组件合计 = Σ(下属零件价格) × 组件数量
电气栏目小计 = Σ(该栏目下属电气零件价格)
```

## 脚本

`scripts/bom-cost-align.py` — 独立 Python 脚本，用法：

```bash
python3 scripts/bom-cost-align.py 11015
python3 scripts/bom-cost-align.py 11719 --dry-run
python3 scripts/bom-cost-align.py project --bom a.xlsx --cost b.xlsx
```

## 已知限制

1. **VGM 件号不匹配**：采购写 `11969VGM-xPxxx`、BOM 写 `11969001-xPxxx`，需人工确认映射规则。
2. **不含调价/利润**：纯采购成本基准。
3. **只用采购数据**：不使用 BOM 原始单价。
4. **无零件级明细不能匹配**：只有品牌/产品汇总金额、没有件号/零件编号的成本表，需要先补零件级成本明细或人工拆分。
5. **重复件号口径**：标准采购明细按件号映射；成本化 BOM 表按同件号出现顺序消费金额，以兼容同一件号多行成本。

## 最小验收

```bash
python3 skills/bom-cost-align/scripts/bom-cost-align.py 11015 --dry-run
python3 skills/bom-cost-align/scripts/bom-cost-align.py 10880 --base /path/to/bom有变形 --dry-run
python3 skills/bom-cost-align/scripts/bom-cost-align.py 11013 --base /path/to/bom有变形 --dry-run
python3 skills/bom-cost-align/scripts/bom-cost-align.py 11090 --base /path/to/bom有变形 --dry-run
python3 skills/bom-cost-align/scripts/bom-cost-align.py 11900 --base /path/to/bom有变形 --dry-run
```

验收点：

- 11015 基准项目 dry-run 指标不回退。
- 10880 可自动识别 `{项目号}_BOM.xlsx` 与 `采购成本.xlsx`，纯机械输出电气 `SKIP`。
- 11013 可识别成本表 `Sheet1` 的表头明细。
- 11090 可识别 `机械` / `电气` 成本化 BOM 表，并按成本化 BOM 公式取价。
- 11900 因成本表缺零件级明细而明确报错，不生成误导性文件。
