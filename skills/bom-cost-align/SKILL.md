# BOM 成本对齐 Skill

## 触发条件

当用户说以下关键词时触发：
- "BOM成本核算" / "对齐BOM" / "BOM匹配采购" / "自动算BOM价格"
- "把采购价填到BOM里" / "生成成本核算表"
- 或在项目文件夹语境中要求"处理BOM"

## 功能概述

输入 **原始 BOM + 采购成本表**，输出 **手工版格式的 BOM 成本核算 Excel**。

## 输入文件

### 必须文件（缺一不可）

1. **原始 BOM 表**（`{项目号}.xlsx`）— 统一22列布局
   - `机械BOM表` sheet：零件行带 `件号`（含 `P` 后缀）+ 组件标题行（`（组件名）×N`）
   - `电气BOM表` sheet：电气零件用 `E{项目号}-{序号}` 格式

2. **采购成本表**（含 `明细` sheet）
   - 数据从第 4 行开始
   - col[0]：`图号/零件号`，col[8]：`含税金额`

## 输出文件

`{项目号}_BOM成本核算_自动生成.xlsx`，含两个 sheet：

| 列 | 标题 | 内容 |
|---|---|---|
| A | 件号 | 零件号或组件标题 |
| B | 零件名称 | 组件名或零件名 |
| C | 型号规格 | 规格型号 |
| D | 说明 | 材料/采购类型 |
| E | 数量 | BOM 用量 |
| F | 品牌 | 品牌 |
| G | 单价 | 采购含税金额 ÷ BOM数量 |
| H | 价格 | 采购含税金额 |
| I | 组件数量 | 仅组件行：标题中 ×N 的数量 |
| J | 合计总价 | 仅组件行：零件价之和 × 组件数量 |

### 格式
- 表头：灰底加粗，下边框加粗
- 组件行：浅蓝底色（#DAEEF3）
- 全表细线边框

## 匹配公式

```
BOM单价 = 采购含税金额 ÷ BOM数量
价格 = 采购含税金额
合计总价 = Σ(下属零件价格) × 组件数量
```

## 已知限制

1. **件号前缀不匹配**：部分项目采购文件使用 `{项目号}VGM-xPxxx`，但 BOM 使用 `{项目号}-xPxxx`——需确认映射规则后才能支持此类项目
2. **不含调价/利润**：输出为纯采购成本基准，不包含人工调价、装配费、利润加成
3. **只用采购数据**：不使用 BOM 原始单价，所有价格从采购表计算

## 脚本

`scripts/bom-cost-align.py` — 独立 Python 脚本，用法：

```bash
# 设置项目根目录后运行
export BOM_BASE="/path/to/projects"
python3 scripts/bom-cost-align.py 12345 --base "$BOM_BASE"

# 只统计不输出
python3 scripts/bom-cost-align.py 12345 --base "$BOM_BASE" --dry-run

# 直接指定文件
python3 scripts/bom-cost-align.py MyProject --bom BOM.xlsx --cost 采购.xlsx
```

## 验证结果

已在 4 个实际项目上验证，匹配率 89%-99%，匹配公式精确到 <1%（剩余差异来自人工调价）。
